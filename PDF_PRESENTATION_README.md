### **Slide 6.6: Data vs Parameters Philosophy** (NEW SLIDE)
**Innovation: Novel validation decision framework**

**Traditional Approach** ‚ùå:
- Existing tools either do NO validation (SPSS, R, GraphPad)
- Or attempt validation on parameters (ineffective)

**Guardian Philosophy** ‚úì:
- **If component accepts RAW DATA ‚Üí Validate assumptions**
- **If component only accepts PARAMETERS ‚Üí Skip validation**

**Guardian Validates**:
- Raw sample data ‚Üí t-test
- Raw data arrays ‚Üí ANOVA
- X, Y values ‚Üí Regression
- Sample measurements ‚Üí CI
- **Reason**: Can check normality, variance, independence

**Guardian Skips**:
- Œ±, power, effect size ‚Üí Parameters
- Pre-calculated mean, SD ‚Üí Summary stats
- Visualizations ‚Üí No test execution
- Design wizards ‚Üí Configuration tools
- **Reason**: No raw data = no assumptions to validate

**Result**: Maximum protection where it matters, zero false positives
